# KOL 付款审批自动核对

把你每天手动做的「KOL 付款审批核对」自动化：检查审批表单字段和所附合同
是否对得上，以及有没有重复提交，最后给出 **PASS / FAIL + 具体原因**。

## 它现在能做什么

把「合同 PDF + 审批内容（截图/PDF/文字）」丢进去，它会：

1. 用 Claude 把合同和审批分别读成结构化字段（只读取，不做计算/判断）；
2. 用**写死的代码**跑下面 19 条核对（金额这类算术绝不让 AI 估）；
3. 做重复提交检测；
4. 输出一份核对报告。

核对的 19 条（含重复检测），全部用真实日常数据校准过：

| # | 检查 | 说明 |
|---|------|------|
| 0 | 重复提交 | 相同 项目/KOL/账户/金额/币种 之前处理过 → 标记重复 |
| 1 | 项目一致 | 审批项目 = 合同 WHEREAS 里的 App 名（不是甲方公司名） |
| 2 | KOL 昵称一致 | 审批 KOL = 合同 KOL（差一两字母转人工；账户+邮箱都对疑似换号转人工）|
| 2b | 合同乙方名称已填写 | 合同「乙方信息」Legal Name 这行必须填，空着转人工 |
| 3 | Ocean Look → PayPal | Ocean Look 项目收款方式必须是 PayPal |
| 4 | 账户名称一致 | 审批收款账户名 = 合同收款账户名（含 PayPal/Payoneer）|
| 4b | 收款人姓名(银行需英文) | 银行收款人姓名须英文；韩/日放行，PayPal/Payoneer 不限 |
| 5 | 币种一致 | 审批币种 = 合同币种 |
| 6 | 金额÷单价=整数视频数 | 审批金额 ÷ 合同单价 必须整除 |
| 7 | 视频数×平台数=合作视频数量 | 实际视频数 × 平台数 = 应填写数量 |
| 8 | 视频清单条数一致 | 视频清单条数 = 合作视频数量 |
| 8b | 视频清单无重复链接 | 同一链接贴两遍 → 标记 |
| 11 | 收款账号核对 | 邮箱 / IBAN / SWIFT / 银行账号 与合同一致（账号被改成别人最危险）|
| 12 | 收款辅助信息 | 国家 / 地址 / 邮编 与合同一致，不一致转人工 |
| 13 | 备注一致性 | 备注里的总额/单价/条数/平台数与字段、合同对得上 |
| 9 | 预付款 | 单独标记，跳过视频数量算术，转人工 |
| 10 | 上线类型 | 非「KOL上线」（社媒采买/SEO上线等）一律转人工 |
| 10c | 纯 YouTube 长视频 | 只发 YouTube = 长视频，上线类型应为「SEO上线」，转人工 |

> 结论分三档：**PASS（通过）/ FAIL（打回，硬错误）/ FLAG（转人工，需人眼确认，不算失败）**。

## 关于「全自动机器人」

你想要的是飞书里自动抓单、自动核对、自动出结论。这套核对引擎是其中**最核心、
最复用的一块**——不管将来数据怎么进来，核对逻辑都是这一套。

要真正做到「自动抓单」，差**一步**：飞书的审批数据要被程序读取，必须由
**管理员**在飞书开放平台建一个「企业自建应用」，并开通审批读取 + 附件下载权限
（普通账号无法绕过，这是飞书的安全设计）。

- ✅ **拿到管理员授权前**：用下面的「半自动模式」——手动下载合同 PDF + 审批截图，
  丢给引擎核对。逻辑完全一样，一天几十单点几下就过完。
- 🔜 **拿到授权后**：把 App ID/Secret 填进环境变量，补全 `feishu/client.py` 里
  几个 TODO（已搭好骨架），即可变成全自动定时跑。

## 用法

### 半自动模式（接 Claude 自动读 PDF/截图，今天就能用）

```bash
export ANTHROPIC_API_KEY=你的key
python -m kol_audit.cli --approval 审批截图.png --contract 合同.pdf
```

`--approval` 支持：审批截图（png/jpg）、审批导出的 PDF，或一段纯文字。

### 离线模式（不调 API，直接喂结构化 JSON，用来验证逻辑）

```bash
PYTHONPATH=src python -m kol_audit.cli \
  --approval-json tests/fixtures/approval_ok.json \
  --contract-json tests/fixtures/contract_ok.json
```

返回码：通过为 0，未通过（含重复提交）为 1——方便接自动化流程。

## 项目结构

```
src/kol_audit/
  models.py    审批 / 合同 数据模型（业务字段契约）
  rules.py     ★ 19 条核对规则，纯函数、可单测，核对逻辑都在这
  dedup.py     重复提交检测（本地 JSON 记指纹）
  extract.py   用 Claude 从 PDF/截图提取结构化字段
  report.py    生成 PASS/FAIL 报告
  cli.py       命令行入口
feishu/
  client.py    飞书审批 API 接入骨架（拿到管理员权限后补全 TODO）
tests/
  test_rules.py  规则单测（44 个用例，覆盖各种通过/失败）
  fixtures/      示例审批/合同 JSON
```

## 设计原则

- **算术和判断用代码，不用 AI**：金额÷单价、数量比对、PASS/FAIL 都在 `rules.py`
  里写死，保证可靠、可复现；Claude 只负责把图文「读成字段」。
- **金额用 Decimal**：避免浮点误差导致的假性「不整除」。
- **字段比对先归一化**：大小写、全角/半角、首尾空格统一，减少假性不一致。
- **数据入口可替换**：手动喂 / 截图识别 / 飞书 API 自动抓，核对逻辑零改动。
