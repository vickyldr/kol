# 网页版：一次拖一堆文件，出一张总表

> 解决「聊天框一次只能传 5 个、特别慢」的问题。网页版一次拖几百个文件都行。

## 是什么

一个网页：把当天**所有**审批单 + 合同的 PDF 一次性拖进去 → 自动判断哪个是审批、
哪个是合同 → 按 KOL 自动配对 → 逐单跑完整 19 条核对 → 出一张总表
（❌ 打回的排最前，带原因；✅ 通过的在后）。

**核对引擎带强制闸门**：每单 19 条规则一条都不能漏，漏了直接报错、出不来结果——
不是凭感觉查。

## 给技术：怎么跑起来（5 分钟）

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=...     # 提取后端；或换成公司内部 OCR/私有模型（见数据安全评估）
uvicorn web.server:app --host 0.0.0.0 --port 8000
```

浏览器打开 `http://localhost:8000`，把文件拖进去，点「开始核对」。
- 想让公司同事都能用：部署到内网一台服务器，大家访问那台机器的 8000 端口即可。

## 业务（非技术）怎么用

1. 打开技术给你的网址；
2. 把当天下载的审批 + 合同 PDF **全选**拖进框里（多少个都行）；
3. 点「开始核对」，等一会出总表；
4. 先看 ❌ 打回的，再扫 ⚠️ 人工的，✅ 通过的可放心。

## 关键说明

- **提取后端可替换**：默认用 Claude 读 PDF（需 `ANTHROPIC_API_KEY`）。
  要数据不出内网，把 `web/server.py` 里的 `extract_approval/extract_contract`
  换成公司内部 OCR/私有模型即可，核对逻辑零改动。见《数据安全与财务敏感性评估.md》。
- **判断审批/合同**在本地完成（读"小额付款申请"标记），不外发。
- **读不出来的文件会单独列出来**提醒重传，绝不静默跳过。

## 为什么这个比「在聊天里发」靠谱

- 聊天里靠人逐批发、人来核 → 可能传错文件、可能漏查规则。
- 网页版：一次全拖进去、按 KOL 自动配对、19 条规则代码强制跑全 → 又快又不漏。
- 再往上一层就是飞书自动版：连"下载文件、上传"都省了，直接从飞书拉审批和附件。
