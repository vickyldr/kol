# Chrome 应用商店「不公开(Unlisted)」上架指南

> 目标：团队装一次，以后你每发新版本，Chrome 自动更新给所有人，零操作。
> 「不公开」= 只有拿到链接的人能装，不会被公开搜索到。

---

## 一、准备（一次性，约 10 分钟）

1. **注册开发者账号**（一次性 $5）
   - 打开 https://chrome.google.com/webstore/devconsole/
   - 用公司/团队的 Google 账号登录
   - 缴 $5 注册费（信用卡，一次性，终身）

2. **生成上传用的 zip**
   - 在 `bilingual-extension/` 目录运行：
     ```bash
     bash build-store-zip.sh
     ```
   - 会生成 `kol-assistant-v0.20.0.zip`（只含插件 13 个文件，不含后端代码）

3. **托管隐私政策**（商店要求）
   - 把本目录的 `隐私政策.md` 内容放到一个公开网址，最简单两种：
     - 传到你们的 GitHub 仓库，开 GitHub Pages，得到一个链接；或
     - 用任意 Notion / 飞书文档「公开分享」得到链接。
   - 记下这个网址，第四步要填。

---

## 二、第一次上架（约 15 分钟 + 审核等待）

1. 进开发者后台 → **「新建项」(New Item)** → 上传刚才的 zip。

2. 填写商店信息（**可直接抄 `商店文案.md`**）：
   - 名称、简介、详细描述
   - **类别**：生产力工具 (Productivity)
   - **语言**：中文（简体）
   - **图标**：已在 zip 内（128px）；商店还要 1~2 张**截图**（1280×800 或 640×400），随便截插件侧边栏 + IG 提醒红点即可。

3. **隐私 (Privacy) 标签页**（重要，决定能不能过审）：
   - **单一用途说明**：填 `商店文案.md` 里的「单一用途」。
   - **权限用途**：逐条填 `商店文案.md` 里「权限说明」（每个权限为什么要）。
   - **数据用途**：勾选「会收集用户的消息文本」，并说明仅发送到团队自建服务器做翻译/提醒，不出售、不用于广告。
   - **隐私政策网址**：填第一步托管的链接。

4. **可见性 (Visibility)** → 选 **「不公开 / Unlisted」**。

5. 提交审核。Unlisted 通常**几分钟到几小时**，偶尔到 1 天。

6. 过审后，后台给你一个**安装链接**（形如 `https://chrome.google.com/webstore/detail/<你的ID>`）。
   把这个链接发给 7-8 个同事 → 他们点「添加到 Chrome」→ 装好。

---

## 三、以后每次更新（你的日常流程）

> 团队那边**什么都不用做**，几小时内自动更新。

1. 改完插件代码后，**把版本号 +1**：编辑 `manifest.json` 的 `"version"`（如 `0.20.0` → `0.20.1`）。
2. 重新打包：`bash build-store-zip.sh`
3. 开发者后台 → 你的项 → **上传新 zip → 提交**。
4. 等审核通过，团队自动更新完成。

**重要区分——很多改动根本不用走这步：**
- 改 **`server.js`**（AI 判断逻辑、提示词、话术、缓存…）→ 只在 VPS 重启服务，**全员秒生效，插件和商店都不用动**。
- 只有改 **插件前端**（`content.js` / `kol-reminder.js` / 侧边栏 / `manifest`）才需要重新发商店。

---

## 四、给同事的一句话说明

> 点这个链接 →「添加到 Chrome」装一次。以后会自动更新，不用再管。
> 第一次用：打开插件侧边栏 → ⚙️ 服务器设置填好地址和口令 → 🔔 提醒里设一下「我的产品」。
