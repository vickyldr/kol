# KOL 双语沟通助手 · 功能说明书（实时维护）

> 这份文档逐个功能讲清楚：**输入什么 → 输出什么 → 解决什么烦恼 → 背后给 AI 问的是什么 → 设计思路 → 和相邻功能的区别**。
> 以后想改某个功能、或要跟别人解释「这个按钮干嘛的」，看这一份就够。
> 维护方式：每次改动功能，顺手更新对应小节。当前对应版本：**v0.22.2**。

---

## 0. 总原则（安全红线）

- **只读不写**：插件只「读」Instagram 网页上已经渲染出来的文字，**绝不调用 IG 接口、绝不替你自动发消息、绝不动你账号**。所有「发送」都是你自己手动点 IG 的发送键。
- **数据存本地**：合作进度、提醒、待办都存在你浏览器的 `chrome.storage.local` 里，不上传服务器。只有「翻译/分析/总结」这类要 AI 算的，才把那段文字发到我们自己的服务器转给千问。
- **扩展 ID 已固定**（v0.22.1 起）：更新插件不会丢本地数据。

## 1. 顶部 · 通用设置

### 1.1 ⚙️ 服务器设置
- **输入**：服务地址（本机 `http://127.0.0.1:3210` 或公司 VPS 地址）、团队口令、管理员口令（选填）。
- **输出**：右上角变「已连接」。
- **解决**：让所有 AI 功能能用；团队共用一台服务器、共用一套话术库。
- **思路**：普通成员填地址+口令即可；填了管理员口令才能编辑/删除话术，防止误删。

### 1.2 当前产品（下拉）
- **输入**：选一个产品（recco / rythmix / aicatch / vivavideo / 通用）。
- **输出**：之后生成回复、分析时，AI 会带上这个产品的卖点和「禁止乱说」的限制。
- **解决**：同一个人对接多个产品时，回复口径不串。
- **思路**：产品资料存服务器，选「通用」就不带品牌细节。

### 1.3 💾 导出备份 / 📂 导入恢复
- **输入**：点导出生成一个 JSON 文件；点导入选一个之前导出的 JSON。
- **输出**：导出＝一个备份文件；导入＝把合作进度/提醒/待办/身份设置合并恢复。
- **解决**：换电脑搬家、或万一数据出意外时还原。
- **思路**：按对话 key 合并，不会用空数据覆盖已有的。

## 2. 主区 · 跟红人沟通（一列）

> 两个输入框，分工明确：
> **第一框「红人发来的原文（外语）」**＝看不懂的、对方发来的话。
> **第二框「我想回复（中文）」**＝你想说的中文 / 你的疑问。

### 2.1 🔤 忠实翻译
- **输入**：第二框写中文（你的成稿）。第一框有红人原文时，用来判断翻成哪国语言。
- **输出**：上方「双语回复」卡——左外语 / 右中文，逐句对照。
- **给 AI 问的**：「把这段中文**一字不改、不润色、不加客套**地翻成目标语言。」
- **解决**：你已经想好怎么说，只要精准翻过去，不要 AI 自由发挥。
- **思路**：温度=0，强调保留主语/否定/数字/语气。目标语言＝你手选的「回复语言」；没选就跟随红人原文语言；红人原文也空就退回你设的「常用语言」。
- **区别**：和「润色生成」的区别＝忠实=照翻，润色=会帮你写得更自然。

### 2.2 📝 润色生成
- **输入**：第二框写中文（成稿**或**只是大概意思，例如「委婉拒绝涨价，给两点理由」）。
- **输出**：同上，双语回复卡。
- **给 AI 问的**：「判断这是成稿还是写作意图，结合红人原话、上下文、产品资料，生成自然、专业、像真人的正式回复。」
- **解决**：你只有个大概想法，让 AI 帮你组织成得体的外语回复。
- **思路**：温度=0.25，允许润色但禁止编造价格/日期/授权/链接。
- **区别**：比忠实翻译「聪明」，但也因此可能加东西——要原样照翻就用忠实翻译。

### 2.3 ❓ 这是什么意思（看懂·含潜台词）
- **输入**：可填可不填。**不填也能用**——它会自动扫描当前对话框。
- **输出**：「🤖 AI 回答」区——大白话逐句讲清在说什么 + **可能的言外之意/潜台词**。
- **给 AI 问的**：`/api/ask`，**永远先用扫描工具读当前对话框**（通常最全），再结合你粘贴的原文；明确告诉 AI「**以信息更全的一方为准**」——通常扫描的更全，但你粘贴里若有扫描没有的内容，就两边结合。
- **解决**：看不懂、或拿不准对方话里藏着什么意思；也防止你粘贴的原文其实不全自己却没察觉。
- **思路**：要「聪明、会多说几句」，所以结合完整上下文解读，而不是干巴巴翻译。
- **区别**：翻**别人发来的、还要解读** → 这个；验**AI 给你的回复** → 看 2.6 的自动逐句对照。

### 2.4 🤔 这怎么办
- **输入**：第二框写你的疑问（例如「这个人死活不同意降价怎么办」）。第一框红人原文会一起带上。
- **输出**：「🤖 AI 回答」区——给思路/建议，**不生成给红人的回复**。
- **给 AI 问的**：`/api/ask`，把你的问题 + 当前对话（自动抓） + 红人原文 + 产品资料一起给 AI 出主意。
- **解决**：拿不准怎么处理，想要个参谋；或想问跟回复无关的问题。
- **思路**：自动读你停在的那段 IG 对话当背景，不用你手动粘。
- **区别**：「这是什么意思」=看懂字面；「这怎么办」=出主意、给策略。

### 2.4b ⚡ 我的快捷回复（个人 · 本地 · 打字秒出）
- **输入**：在搜索框打几个字（触发词或内容里的词）。
- **输出**：即时蹦出你存的快捷条目，点一条 → 直接填进双语回复卡。
- **怎么存**：① 生成回复后点卡片上「⭐ 存为快捷」，填个触发词即可；② 或「＋ 手动加一条」自己写。
- **解决**：每个人都有自己高频用的几十条话术，不该每次去翻团队大库；打几个字秒出最省事。
- **思路**：**纯本地关键词匹配，不走 AI、不走服务器** → 准、即时、离线也能用。存在浏览器 `kolQuickReplies`，跟着「💾导出备份」一起备份。
- **区别**：团队话术库＝共享、权威、配置在服务器；这个＝个人、私有、存本地、求快。

### 2.5 📖 从话术库选回复话术（团队共享库）
- **输入**：点开，搜/选一条已存好的话术。
- **输出**：套进双语回复卡。
- **解决**：高频场景不用每次重写，直接调现成话术。

### 2.6 双语回复卡（结果区 · 自动逐句对照）
- **左右分屏，每句自动对齐**：左外语 | 右中文，**每次生成回复都自动逐句对齐**（后台 `/api/align`，保证一句外语对一句中文），**不用按任何按钮**。
  - 先用按标点的快速切分瞬间显示，随后自动替换成对齐可信的版本。
  - **鼠标移到一行** → 两边对应那句一起高亮，一眼知道哪句对哪句。
  - **点左边外语** → 把整行标记住（持久高亮），方便回头挑句。
  - **点「复制」**（每行右侧，悬停出现）→ 只复制这一句外语。
- **改在中文这边**（你不懂外语，外语只读）：
  - **右边中文格可以直接改**（你自己改意思）；
  - 改完点 **「✅ 按修改后的中文改写外语」** → AI **重新改写**外语来匹配你的中文（是改写润色、不是死板直译），再自动逐句对齐。
- **复制全部外语**：一键复制整条，去 IG 粘贴发送。
- **保存为话术**：把这条存进话术库复用。
- **回复语言**：默认「自动（跟随红人语言）」，也可手选英语/韩语/泰语等。
- **或：写一句话让 AI 改**：不想自己改中文、只想给个指令时用，写「太长精简点 / 语气热情点 / 别提价格」→「让 AI 改」。
- **解决**：① 不确定 AI 翻得对不对 → 对照一眼看穿哪句翻歪；② 回复有两三句、你只想用其中一句 → 知道哪句外语对应哪句中文，单独复制那一句。

### 2.7 识别 & 内部提醒（折叠在下方 · 后台加载）
- **触发**：第一框有红人原文时，点忠实翻译/润色生成后自动后台分析并展开。
- **输出**：识别语言、对话阶段、对方意图、可能的言外之意、备选回复、风险提示、**要不要先问 TL**、等待期可先发的临时回复、对方提到的物料、还缺哪些信息。
- **给 AI 问的**：`/api/analyze`，对红人这条消息做深度理解。
- **解决**：新人吃不准对方话里的潜台词、不知道要不要请示 TL。
- **思路**：和回复分两段——回复 3-5 秒先出，分析慢几秒补上，不挡你干活。
- ⚠️ 紫色「内部处理建议」是给你自己看的，**别发给红人**。

## 3. 当前合作情况（进度清单）

- **输入**：点「📥 抓取这屏」把当前可见对话拉进来（可多次滚动累积），或自己粘贴；点「🔄 总结这段对话」。
- **输出**：一张一眼能扫的清单——✅已做 / ⬜没做 / ⚠️备注（合同发了没、积分冲了没、brief 发了没、初稿收了没……）。
- **给 AI 问的**：`/api/summary`，把对话整理成固定顺序的 12 步流程清单；带上已有进展做**增量更新**，不覆盖旧的。
- **解决**：聊到一半忘了走到哪、还差啥；过几天想不起来「到底给 Ta 发合同没」。
- **思路**：
  - 存在 `kolSummaries`，**按对话固定 ID 存**（v0.21.9 起），名字识别有出入也不丢匹配。
  - 你切到别的对话时**自动后台更新**这张清单。
  - 「✅此合作已完结」→ 把这个对话静音，不再提醒。
- **区别**：这是「整段关系的进度」；提醒是「该我做某件事的待办」。

## 4. 🔔 提醒

> 三类提醒，全部**只读 DOM 判断**，不碰 IG，不替你回。

### 4.1 身份设置（认我方 vs 红人）
- **输入**：你的产品前缀（recco, rythmix, aicatch, vivavideo）、你自己的 handle。
- **输出**：插件能区分「我方同事」（前缀开头）和「红人」。
- **解决**：群里同事已回 ≠ 你已处理；只有**你自己**回了才消提醒。
- **思路**：蓝色/右对齐气泡＝「我」；前缀开头＝同事；其余＝红人。

### 4.2 待回复 / 待跟进 / 待办
- **待回复**：红人发来了、你没回（收件箱「N 条新消息」/蓝点判断）。
- **待跟进**：口头答应了却没推进（要 id/合同/初稿等），AI 按话术流程节点判断。
- **待办**：你手动加的事，或 AI 自动建议的。
- **给 AI 问的**：`/api/judge`，按话术流程节点表判断该不该提醒、卡在哪一步。
- **输出**：插件图标小红点 + 桌面通知 + 清单。
- **思路**：闹钟 1 小时跑一次；寒暄不用回的会自动忽略，不瞎提醒。

### 4.3 智能加待办
- **输入**：打一句话，例如「5 天后小树枝交稿 / 明天下午 3 点催 Anna」。
- **输出**：自动解析成「事项 + 日期 + 时间」存进待办。
- **给 AI 问的**：`/api/parse-todo`，自然语言 → 结构化时间。

### 4.4 弹出清单窗口 / 启动弹窗
- **解决**：你懒得开侧边栏。开 Chrome 时若有当天该办的，自动弹一个 todo 小窗口。

## 5. 后台 · 服务端（给懂技术的人看）

### 5.1 接口一览
| 接口 | 干什么 | 用的模型 |
|---|---|---|
| `/api/reply` | 快出双语回复（3-5 秒） | 快（qwen-flash） |
| `/api/analyze` | 深度分析红人消息 | 快 |
| `/api/rewrite` | 翻译/润色/改写（faithful / chinese_to_target / refine / target_to_chinese） | 快 |
| `/api/translate` | 纯翻译 + 难词注释 | 快 |
| `/api/ask` | 问 AI / 看懂 / 出主意 | 快 |
| `/api/judge` | 提醒判断（该不该提醒、卡哪步） | 慢（qwen-plus/3.7） |
| `/api/summary` | 合作进度 12 步清单（增量更新） | 快 |
| `/api/parse-todo` | 自然语言 → 定时待办 | 快 |

### 5.2 模型路由
- **用户面前要等结果的，全用快模型**（qwen-flash），追求速度。
- **只有后台默默判断提醒**用慢模型（更准，反正不用人等）。
- 每个模型有各自独立的免费额度。

### 5.3 缓存
- `translateCache`（24 小时）、`judgeCache`（1 小时）：同样的文字不重复花钱、更快。

## 6. 数据存放与边界（拆仓库 / 换电脑 / 分享前必看）

> 一句话：**代码在 GitHub，话术/物料/知识库（线上版）在 VPS 服务器，进度/提醒在各人浏览器。三条线互不相干。** 拆代码仓库只动第一条，后两条纹丝不动。

| 资料 | 存在哪 | 在 git 里吗 | 拆仓库 / 更新代码会动它吗 |
|---|---|---|---|
| 插件代码、文档 | GitHub 仓库 `bilingual-extension/` | ✅ 在 | 跟着代码走（应该的） |
| **出厂种子话术**：`data/playbook.json`、`knowledge-base.json`、`quick-templates.json` | 仓库 `data/` | ✅ 在 | 跟着代码走（这是"装上就带的默认话术"） |
| **团队在插件里新增的话术存档、物料库** | VPS `~/kol-data/`（`scenario-archive.json`、`assets/`） | ❌ 不在 | ❌ 不动 |
| **你单独维护 / 多项目的知识库、话术脚本（线上覆盖版）** | VPS `~/kol-data/`（同名文件覆盖种子） | ❌ 不在 | ❌ 不动，更新代码也不覆盖 |
| 产品资料 `products.json` | VPS `~/kol-data/` | ❌ 不在 | ❌ 不动 |
| 各人**合作进度 / 提醒 / 待办 / ⚡个人快捷回复** | 各自浏览器 `chrome.storage.local`（含 `kolQuickReplies`） | ❌ 不在 | ❌ 不动（连服务器都不在；可用「💾导出备份」搬家） |

**放哪里 = 看你要什么效果（这就是区分规则）：**

- 想让一份话术**「跟着代码走、所有人装插件都带上、随版本更新」** → 放仓库 `data/`（出厂种子）。
- 想让它**「线上随时改、团队共享、不进代码、更新代码时不被覆盖」** → 放 VPS `~/kol-data/`。

**服务器怎么自动区分（`seedOrLive`）**：读 `playbook/knowledge-base/quick-templates` 时，
**先看 `~/kol-data/` 里有没有同名文件**——有就用线上那份（你的覆盖版），没有才用代码种子。
所以你以后要更新其它项目 / 单独维护的知识库，**只放到 VPS `~/kol-data/`，永远不会污染代码仓库、也不会被代码更新冲掉**。改完在 VPS 重启一下服务即可生效。

**关于拆仓库**：拆出去的只是 `bilingual-extension/` 这份代码（含 `data/` 种子）。
团队天天用产生的话术、物料、进度全在上面那两条「不在 git」的线里，**一个都不会被挪走**。
插件打包的 zip 里**也不含 `server.js` 和 `data/`**——同事的浏览器插件不携带任何话术，运行时才从服务器取。

## 7. 待定 / 讨论中

- **三个「翻译/验证」入口已厘清**（v0.22.4 定）：
  - 看对方发来的 + 解读潜台词 → 「❓这是什么意思」（结合上下文，聪明版）；
  - 验 AI 给你的回复 → 双语回复卡**自动逐句对照**（无需按钮，悬停高亮、单句可复制）；
  - 页面里逐条自动翻译 → 字面、被动、挂在消息旁。
